Tutorial: Configuring a Scene (NYC Example)

This tutorial explains how to configure a scene prefab after it has been generated from a .vox file. The concepts here apply to any scene, such as a city environment like "NYC".

Default Configuration

When you generate a prefab for a Scene, it typically comes with VoxelObjectProxy and RigidbodyProxy components pre-attached. These proxies simplify the complex setup required for voxel physics and rendering.

VoxelObjectProxy

The VoxelObjectProxy acts as a high-level wrapper for the core voxel systems. It is effectively equivalent to combining:

  • ChunkBasedMarchingCube (Rendering)
  • ChunkBasedMarchingCubeSavingSystem (Persistence)
  • PxVoxelColliderGenerator (Collision)

Proxy Types

You can configure the behavior of the voxel object by setting the VoxelObjectProxyType. The available types are:

public enum VoxelObjectProxyType
{
    SceneStatic = 0,          // Non-moving environment parts (e.g., ground, buildings)
    SceneDynamic,             // Moving environment parts
    SceneStrongConnected,     // Strongly connected structures
    SceneWeakConnected,       // Weakly connected structures
    WeaponItem,               // For weapons
    BuildingItem,             // For construction items
}

RigidbodyProxy

The RigidbodyProxy enables physics interactions and destruction. It is equivalent to combining:

  • UnityEngine.Rigidbody
  • VoxelCollider
  • PxConverter

This component allows voxel objects to have physical weight, respond to gravity/collisions, and support destruction events.

Advanced Configuration

If you prefer granular control and understand the underlying parameters, you can opt out of using the Proxies.

  1. Manual Component Setup: You can manually attach ChunkBasedMarchingCube, PxVoxelColliderGenerator, etc., instead of the Proxy.
  2. Physics Components: When manually configuring physics, use standard UnityEngine.Physics components. Do not attach Px5 specific components directly; the mod loader will automatically assemble the necessary PxConverter at runtime.
